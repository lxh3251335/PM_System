## 🆕 新增页面详情

### 1️⃣ 设备关系配置页面（device-relations.html）

**页面功能：**
- 配置温控器与冷风机的关系
- 配置机组与冷风机的关系
- 配置电表与用电设备的关系
- 配置智能融霜控制器与冷风机的关系

**关系类型：**
| 关系类型 | 源设备 | 目标设备 | 说明 |
|---------|--------|---------|------|
| 温控器 → 冷风机 | 温控器 | 冷风机 | 温控器控制冷风机的启停，一般一个冷库配置一个温控器 |
| 机组 → 冷风机 | 机组 | 冷风机 | 机组为冷风机供冷，一台机组可以为多台冷风机供冷 |
| 电表 → 用电设备 | 电表 | 机组/冷风机 | 电表监测机组、冷风机等用电设备的电量消耗 |
| 融霜控制器 → 冷风机 | 智能融霜控制器 | 冷风机 | 智能融霜控制器控制冷风机的融霜过程 |

**界面特点：**
- ✅ 关系类型切换选项卡
- ✅ 关系统计（实时显示已配置关系数）
- ✅ 下拉选择源设备和目标设备
- ✅ 关系列表可视化展示（设备A → 设备B）
- ✅ 支持删除关系
- ✅ 详细的关系说明和提示

**操作流程：**
```
1. 选择关系类型（如：温控器 → 冷风机）
2. 从下拉框选择温控器（如：TC-001-001）
3. 从下拉框选择冷风机（如：AC-001-001）
4. 点击"添加关系"
5. 系统显示：TC-001-001 → AC-001-001（温控器控制冷风机）
6. 继续添加其他关系
```

### 2️⃣ 通讯配置及485地址分配页面（communication-setup.html）

**页面功能：**
- 为网关的每个RS485端口分配通讯设备
- 为网关的每个网口端口分配通讯设备
- 为每个设备分配唯一的485从站地址（1-247）

**核心功能：**
| 功能 | 说明 |
|------|------|
| 端口可视化 | 网格显示所有RS485端口和网口端口 |
| 端口状态 | 清晰区分"已分配"和"未分配"端口 |
| 设备分配 | 点击端口打开弹窗，选择通讯设备 |
| 485地址管理 | 为每个设备分配1-247之间的唯一地址 |
| 地址冲突检测 | 自动检测485地址是否已被使用 |
| 设备过滤 | 只显示具有通讯功能的设备 |
| 配置预览 | 实时显示端口分配情况 |

**界面特点：**
- ✅ 统计卡片（已分配端口/总端口数/已分配设备/通讯设备总数）
- ✅ 按网关分组显示
- ✅ RS485端口和网口端口分别展示
- ✅ 端口状态颜色区分（未分配：灰色，已分配：蓝色）
- ✅ 弹窗配置界面（选择设备、输入485地址）
- ✅ 地址唯一性校验
- ✅ 清除配置功能

**操作流程：**
```
1. 查看网关信息（GW-001-001，华为 AR151，RS485:8口 网口:2口）
2. 点击"RS485-1"端口
3. 弹窗打开，选择设备：TC-001-001（精创 STC-200+ 温控器）
4. 输入485地址：1
5. 点击"保存配置"
6. 端口变为蓝色，显示：TC-001-001 | 485地址: 1
7. 继续配置其他端口
```

**485地址分配示例：**
```
RS485-1: TC-001-001（温控器）   485地址: 1
RS485-2: AC-001-001（冷风机）   485地址: 2
RS485-3: AC-001-002（冷风机）   485地址: 3
RS485-4: UN-001-001（机组）     485地址: 4
RS485-5: PM-001-001（电表）     485地址: 5
...
```

## ❓ 常见问题

### Q1: 为什么要分两步配置（设备关系 + 通讯配置）？
**A**: 这是两个不同的概念：
- **设备关系**：业务逻辑层面的关系（温控器控制哪些冷风机）
- **通讯配置**：物理连接层面的配置（哪个设备接在哪个端口，485地址是多少）

### Q2: 485地址的范围为什么是1-247？
**A**: 这是Modbus RTU协议的标准规定：
- 0地址为广播地址
- 1-247为有效从站地址
- 248-255为保留地址

### Q3: 通讯设备如何识别？
**A**: 设备必须同时具有以下两个字段才会被识别为通讯设备：
- `comm_port`（通讯端口，如 RS485）
- `comm_protocol`（通讯协议，如 Modbus RTU）

### Q4: 可以跳过某些步骤吗？
**A**: 
- 步骤4（设备关系）：可以跳过，但建议至少配置温控器与冷风机的关系
- 步骤6（通讯配置）：可以跳过，但如果配置了网关，建议完成通讯配置
