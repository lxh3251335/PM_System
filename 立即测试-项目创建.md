## 🐛 如果测试失败

### 失败场景1：提示"网络错误"或"请求超时"

**原因：** 后端服务未运行

**解决：**
```bash
cd D:\projects\PM_System\backend
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

---

### 失败场景2：提示"CORS错误"

**Console显示：**
```
Access to fetch at 'http://localhost:8000/api/projects' from origin 'null' has been blocked by CORS policy
```

**原因：** `.env`文件的CORS配置不正确

**解决：**
1. 检查 `backend/.env` 文件
2. 确认 `CORS_ORIGINS` 包含 `"null"`：
   ```
   CORS_ORIGINS=["http://localhost:3000","http://localhost:5173","http://localhost:8000","http://127.0.0.1:8000","http://localhost:5500","http://127.0.0.1:5500","null"]
   ```
3. 重启后端服务

---

### 失败场景3：提示"字段验证失败"

**可能的错误信息：**
- "field required"（缺少必填字段）
- "value is not a valid date"（日期格式错误）

**原因：** 必填字段未填写，或格式不正确

**解决：**
1. 确保所有带 `*` 的字段都已填写
2. 日期格式必须是：YYYY-MM-DD（如：2026-03-01）
3. 电话号码不要有空格或特殊字符

---

### 失败场景4：后端返回500错误

**Network显示：**
```
POST /api/projects
Status: 500 Internal Server Error
```

**原因：** 后端代码错误或数据库问题

**解决：**
1. 查看后端终端窗口的错误日志
2. 将完整的错误信息截图或复制下来
3. 联系开发人员排查
