### Bug #1: CORS配置不支持本地文件访问

**问题描述：**
- 前端通过 `file://` 协议打开时，后端CORS拒绝请求
- 后端日志显示：`OPTIONS /api/projects HTTP/1.1" 400 Bad Request`
- 导致前端无法调用任何API接口

**影响范围：**
- 所有API调用失败
- 无法获取项目列表、创建项目等

**根本原因：**
- `backend/app/config.py` 的 CORS_ORIGINS 只配置了 `localhost:3000` 和 `localhost:5173`
- 没有包含 `null`（file:// 协议的 origin）
- 没有包含常用的开发端口（如5500等）

**修复方案：**
1. 修改 `backend/app/config.py`，增加更多允许的来源：
   - `http://localhost:5500`（Live Server默认端口）
   - `http://127.0.0.1:5500`
   - `http://localhost:8000`
   - `http://127.0.0.1:8000`
   - `"null"`（支持 file:// 协议）

2. 同步更新 `backend/.env` 文件的 CORS_ORIGINS 配置

**修复文件：**
- `backend/app/config.py` (第24-33行)
- `backend/.env` (CORS_ORIGINS配置项)

**验证方法：**
- 重启后端服务（uvicorn会自动reload）
- 打开浏览器开发者工具，查看Network标签
- 确认OPTIONS预检请求返回200 OK
- 确认GET/POST请求正常返回数据

### Bug #3: .env文件编码错误导致后端崩溃

**问题描述：**
- 用户创建项目失败，提示保存失败
- 排查发现后端服务无法启动
- 根本原因：`.env`文件编码错误

**影响范围：**
- 整个后端服务崩溃
- 所有API调用失败
- 用户无法使用任何功能

**根本原因：**
- 使用PowerShell修改`.env`文件时产生了编码问题
- Windows GBK编码与Python UTF-8要求冲突
- 中文注释导致 `UnicodeDecodeError`

**错误信息：**
```
UnicodeDecodeError: 'utf-8' codec can't decode bytes in position 218-219: invalid continuation byte
```

**修复方案：**
1. 删除损坏的`.env`文件
2. 创建Python脚本`create_env.py`用UTF-8编码写入配置
3. 重新启动后端服务

**修复文件：**
- `backend/.env` (重新创建)
- `backend/create_env.py` (新增工具脚本)

**验证方法：**
```bash
cd D:\projects\PM_System\backend
python create_env.py
uvicorn app.main:app --reload
```
- 确认后端成功启动
- 无UnicodeDecodeError错误

### Bug #4: 前后端字段名不匹配

**问题描述：**
- 即使后端启动成功，创建项目仍然失败
- 期望到货时间无法正确保存
- 项目列表不显示到货时间

**影响范围：**
- 项目创建功能
- 项目编辑功能
- 项目列表显示

**根本原因：**
- 前端发送字段名：`expected_arrival_date`
- 后端期望字段名：`expected_arrival_time`
- 字段名不一致导致数据验证失败

**错误的字段映射：**
| 前端发送 | 后端期望 | 状态 |
|---------|---------|------|
| `expected_arrival_date` | `expected_arrival_time` | ❌ 不匹配 |
| `location` | (不存在) | ❌ 多余字段 |
| `status: 'pending'` | (自动设置) | ❌ 不应手动设置 |

**修复方案：**
1. 修改`project-create.html`，统一使用`expected_arrival_time`
2. 移除`location`字段（前端自定义的，后端不需要）
3. 移除`status`字段（后端会自动设置默认值）
4. 修改`project-list.html`，渲染时使用正确字段名

**修复文件：**
- `demo/project-create.html` (第424-439行, 第411行)
- `demo/project-list.html` (第592行, 第754行, 第770行)

**代码修改：**
```javascript
// 修复前
const projectData = {
    expected_arrival_date: formData.get('expected_arrival_time'),  // ❌
    location: `${city} ${address}`,  // ❌
    status: 'pending'  // ❌
};

// 修复后
const projectData = {
    expected_arrival_time: formData.get('expected_arrival_time'),  // ✅
    // 移除location和status
};
```

**验证方法：**
- 打开`project-create.html`
- 填写完整项目信息
- 点击"创建项目"
- 应该成功创建并跳转
