## 二、具体变更内容

### 变更1：新增项目复制功能 🆕

#### 需求描述
用户在创建新项目时，如果历史上有相似项目（设备数量及品牌相似），可以复制该项目的所有信息，然后只修改差异部分，快速完成新项目登记。

#### 业务价值
- **提升效率**：相似项目可节省80%的录入时间
- **减少错误**：复制已验证的数据，降低录入错误
- **标准化**：形成项目模板，统一配置标准

#### 功能设计

**复制范围**：
- ✅ 项目基础信息（项目类型、城市可修改）
- ✅ 冷库信息（名称、类型、面积、高度）
- ✅ 所有设备信息（冷风机、温控器、机组、电表、冷柜）
- ❌ 不复制：项目流程状态、创建时间、项目编号

**操作流程**：
```
1. 用户在项目列表点击"复制项目"按钮
2. 系统弹出确认对话框："是否复制项目【项目名称】的所有信息？"
3. 用户确认后，系统创建新项目（自动生成新编号）
4. 复制所有冷库和设备信息
5. 跳转到新项目编辑页面，用户修改差异信息
6. 保存后完成
```

**数据处理**：
- 项目编号：重新生成
- 项目名称：自动添加"（副本）"后缀
- 设备编号：重新生成（按新项目ID生成）
- 创建时间：使用当前时间
- 创建人：使用当前登录用户
- 流程状态：重置为"新项目"

#### 技术实现
- 后端：添加 `POST /projects/{project_id}/copy` 接口
- 前端：项目列表增加"复制"按钮

#### 优先级
**P0** - 一期必须实现

### 变更2：简化权限管理 🔄

#### 原设计
- 2级权限体系（系统管理员、企业管理员）
- 动态用户创建
- 复杂的权限矩阵
- 企业数据隔离

#### 新设计（V1.1）
- **固定2个用户**：
  1. **用户方账号**：用户名/密码固定（如 user001/password）
  2. **厂家方账号**：用户名/密码固定（如 factory001/password）

- **权限简化**：
  - 两个用户看到相同的数据
  - 无需用户管理功能
  - 无需复杂的权限控制
  - 仅记录操作人（用于日志）

#### 影响范围
✅ **简化功能**：
- 去掉用户管理模块
- 去掉用户注册功能
- 去掉权限检查逻辑

✅ **保留功能**：
- 登录/登出
- 操作日志（记录是用户方还是厂家方操作）

#### 数据模型调整
- **保留users表**：只存储2条固定用户记录
- **去掉role字段复杂枚举**：简化为 user / factory
- **去掉enterprise_name等字段**

#### 技术实现
- 数据库初始化时创建2个固定用户
- 登录验证简化（用户名密码验证即可）
- 去掉所有权限检查中间件

#### 优先级
**P0** - 一期按此简化设计

### 变更3：增加货物到货时间 🆕

#### 需求描述
在项目信息中增加"货物期望到货时间"字段，方便所有人查看项目进度和到货时间。

#### 业务价值
- **进度可视**：一目了然知道货物何时到
- **提前准备**：现场可以提前做好接收准备
- **统一协调**：多个项目到货时间一起看，便于资源调度

#### 字段设计

**项目表新增字段**：
- `expected_delivery_date`：货物期望到货时间（DATE类型）
- 必填项，创建项目时必须填写
- 支持后期修改

**显示位置**：
- 项目列表：增加"期望到货时间"列
- 项目详情：基础信息区域显示
- 项目列表支持按到货时间排序

**筛选功能**：
- 今天到货的项目
- 本周到货的项目
- 逾期未到货的项目（如果流程还在"网关邮寄"之前）

#### 技术实现
- 数据库：projects表增加字段
- 后端：Schema增加字段验证
- 前端：表单增加日期选择器

#### 优先级
**P0** - 一期必须实现

### 变更4：简化设备关系管理 🔄

#### 原设计
- 设备之间建立复杂的关系（冷风机-机组、冷风机-温控器等）
- 单独的关系配置页面
- 关系可视化展示

#### 新设计（V1.1）

**一期（MVP）**：
- ❌ **不建立设备关系**
- 冷风机、温控器、机组、电表、冷柜与冷库**平级关系**
- 数据结构简化：
  ```
  项目
    ├── 冷库1
    ├── 冷库2
    ├── 冷风机1
    ├── 冷风机2
    ├── 温控器1
    ├── 机组1
    ├── 电表1
    └── 冷柜1
  ```

**二期（扩展）**：
- ✅ 再考虑设备关系配置
- ✅ 关系可视化

#### 数据模型调整

**简化前**：
- 设备属于冷库（device.room_id）
- 设备之间有关系（device_relationships表）

**简化后**：
- 设备属于项目（device.project_id）
- 可选关联冷库（device.room_id，可为空）
- 去掉device_relationships表

#### 页面调整

**项目详情页面结构**：
```
项目详情
├── Tab1: 基础信息
├── Tab2: 冷库列表
├── Tab3: 设备管理
│   ├── 冷风机列表
│   ├── 温控器列表
│   ├── 机组列表
│   ├── 电表列表
│   └── 冷柜列表
├── Tab4: 网关管理
└── Tab5: 流程进度
```

**设备添加流程**：
1. 进入项目详情
2. 点击"设备管理" Tab
3. 选择设备类型（冷风机/温控器/机组/电表/冷柜）
4. 点击"添加"按钮
5. 填写设备信息（可选择关联到某个冷库）
6. 保存

#### 技术实现
- 数据库：去掉device_relationships表
- 数据库：devices表的room_id改为可空
- 后端：去掉关系相关API
- 前端：去掉关系配置页面

#### 优先级
**P0** - 一期按简化设计实现

### 变更6：前端UI设计要求 🎨

#### UI/UX设计原则

**采用现代化UI设计风格**：

1. **简洁清爽**
   - 扁平化设计，去除多余装饰
   - 大量留白，信息层次清晰
   - 卡片式布局

2. **专业商务**
   - 深色系配色（深蓝/深灰为主）
   - 高对比度，易读性强
   - 图标统一风格

3. **响应式布局**
   - 支持桌面端（主要）
   - 兼容平板查看

4. **交互友好**
   - 操作有反馈（Loading、Message）
   - 表单验证实时提示
   - 确认操作有二次确认

#### 推荐配色方案

**主色调**：
- Primary: `#1890ff` (蓝色，Element Plus默认主题)
- Success: `#52c41a` (绿色)
- Warning: `#faad14` (橙色)
- Danger: `#f5222d` (红色)

**辅助色**：
- 背景色: `#f0f2f5` (浅灰)
- 卡片背景: `#ffffff`
- 边框色: `#d9d9d9`
- 文字主色: `#262626`
- 文字副色: `#8c8c8c`

#### UI组件使用建议

**Element Plus组件优先**：
- 布局：el-container、el-row、el-col
- 表格：el-table（斑马纹、边框、分页）
- 表单：el-form（标签右对齐、验证提示）
- 按钮：el-button（主按钮/次按钮）
- 对话框：el-dialog（居中、圆角）
- 步骤条：el-steps（流程进度）
- 日期选择：el-date-picker

#### 页面布局建议

**整体布局**：
```
+----------------------------------+
| Header (导航栏 + 用户信息)          |
+--------+-------------------------+
| Aside  |                         |
| (菜单) |  Main Content           |
|        |  (路由内容区)             |
|        |                         |
+--------+-------------------------+
```

**列表页面**：
- 顶部：面包屑导航
- 搜索区：表单形式，支持展开/收起
- 操作区：主按钮（新建）在右上角
- 表格区：斑马纹、分页
- 操作列：编辑、删除、复制等图标按钮

**表单页面**：
- 卡片布局
- 标签右对齐
- 必填项用红色*标注
- 底部按钮：保存、取消

#### 参考设计
- Ant Design Pro（参考布局）
- Element Plus Admin（参考组件使用）
- 追求专业、简洁、高效的视觉体验

#### 优先级
**P0** - 按此UI标准实现
