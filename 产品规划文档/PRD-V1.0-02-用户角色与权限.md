## 一、用户角色定义

### 1.1 角色层级

```
系统管理员（我方）
    └── 企业管理员（客户方）
```

### 1.2 角色详细说明

#### 角色1：系统管理员（Super Admin）
- **描述**：冷库设备集成商的管理人员，拥有系统最高权限
- **典型用户**：技术总监、项目经理、工程师
- **使用频率**：每日多次

#### 角色2：企业管理员（Enterprise Admin）
- **描述**：使用冷库的企业方管理人员，仅能管理本企业的项目
- **典型用户**：企业运营经理、设备维护人员
- **使用频率**：每周数次

### 2.2 数据权限说明

#### 系统管理员
- **可见范围**：所有项目、所有设备、所有用户
- **操作范围**：无限制
- **特殊权限**：
  - 维护标准设备库
  - 创建和管理企业管理员账号
  - 查看所有企业的项目数据

#### 企业管理员
- **可见范围**：仅本企业创建的项目及相关设备
- **操作范围**：仅能操作本企业的项目和设备
- **限制**：
  - 无法查看其他企业的项目
  - 无法修改标准设备库
  - 无法创建其他用户

## 三、用户账号管理

### 3.1 账号创建流程

#### 系统管理员账号
1. 初始部署时由开发人员创建第一个超级管理员账号
2. 超级管理员可创建其他系统管理员账号（可选）

#### 企业管理员账号
1. 系统管理员登录系统
2. 进入"用户管理"模块
3. 点击"新建用户"
4. 填写信息：
   - 用户名（唯一）
   - 密码（首次登录需修改）
   - 企业名称
   - 联系人姓名
   - 联系电话
   - 邮箱
5. 保存后生成账号

### 3.2 账号字段定义

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| user_id | 整数 | 自动 | 用户唯一ID |
| username | 字符串(50) | 是 | 登录用户名，唯一 |
| password | 字符串 | 是 | 加密存储 |
| role | 枚举 | 是 | super_admin / enterprise_admin |
| enterprise_name | 字符串(100) | 条件 | 企业管理员必填 |
| contact_name | 字符串(50) | 是 | 联系人姓名 |
| contact_phone | 字符串(20) | 是 | 联系电话 |
| email | 字符串(100) | 否 | 邮箱地址 |
| status | 枚举 | 是 | active / disabled |
| created_at | 时间戳 | 自动 | 创建时间 |
| last_login | 时间戳 | 自动 | 最后登录时间 |

### 3.3 登录与认证

#### 登录流程
1. 用户输入用户名和密码
2. 系统验证账号状态（是否禁用）
3. 验证密码是否正确
4. 生成会话Token（JWT）
5. 返回用户信息和角色
6. 根据角色跳转到对应首页

#### 会话管理
- Token有效期：24小时
- Token刷新机制：活动状态下自动刷新
- 退出登录：清除Token
- 并发登录：支持同一账号多设备登录

## 四、操作日志记录

### 4.1 日志记录范围
所有涉及数据修改的操作均需记录日志：
- 用户登录/登出
- 创建/编辑/删除项目
- 创建/编辑/删除设备
- 建立设备关系
- 配置网关
- 更新项目流程

### 4.2 日志字段
| 字段 | 说明 |
|------|------|
| log_id | 日志唯一ID |
| user_id | 操作用户ID |
| username | 操作用户名 |
| action | 操作类型（create/update/delete） |
| module | 模块名称（project/device/gateway等） |
| target_id | 操作对象ID |
| description | 操作描述 |
| ip_address | 操作IP |
| created_at | 操作时间 |

## 五、安全策略

### 5.1 密码策略
- 长度：至少8位
- 复杂度：包含字母和数字（一期简化要求）
- 加密：使用bcrypt加密存储
- 首次登录：企业管理员首次登录需强制修改密码

### 5.2 会话安全
- 使用HTTPS加密传输
- Token存储在HttpOnly Cookie中
- 敏感操作需二次验证（如删除项目）

### 5.3 防护机制
- 登录失败5次后锁定账号30分钟
- SQL注入防护（使用ORM）
- XSS防护（前端输入过滤）
