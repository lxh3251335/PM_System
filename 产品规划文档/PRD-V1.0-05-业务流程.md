## 一、核心业务流程图

### 1.1 项目全生命周期流程

```
开始
  ↓
[1. 新建项目] → 填写项目基础信息
  ↓
[2. 设备登记] → 创建冷库 → 添加设备
  ↓

  ↓
[5. 网关登记] → 记录网关信息
  ↓
[6. 网关邮寄] → 更新邮寄状态
  ↓

  ↓
[8. 平台上线] → 系统上线运行
  ↓
[9. 功能确认及完成] → 项目完成
  ↓
结束
```

## 二、详细业务流程

### 流程1：用户登录

**参与角色**：系统管理员、企业管理员

```
用户访问系统
  ↓
输入用户名和密码
  ↓
系统验证
  ├── 验证失败 → 提示错误（失败次数+1）
  │                ↓
  │            失败5次 → 锁定账号30分钟
  │
  └── 验证成功 → 生成Token
                  ↓
              检查是否首次登录
              ├── 是 → 强制修改密码
              └── 否 → 跳转到首页
```

**详细步骤**：
1. 用户打开系统登录页面
2. 输入用户名和密码，点击"登录"
3. 系统验证账号状态（是否禁用）
4. 系统验证密码是否正确
   - 如果错误，提示"用户名或密码错误"，记录失败次数
   - 如果连续失败5次，锁定账号30分钟
5. 验证成功，生成JWT Token
6. 检查 need_change_password 字段
   - 如果为1（首次登录），跳转到修改密码页面
   - 如果为0，跳转到首页
7. 记录操作日志（登录时间、IP地址）

### 流程2：新建项目

**参与角色**：系统管理员、企业管理员

```
用户点击"新建项目"
  ↓
填写项目信息表单
  ├── 项目类型（下拉选择）
  ├── 城市（下拉搜索）
  ├── 项目名称
  ├── 项目地址
  ├── 邮寄地址
  ├── 邮寄联系人
  ├── 邮寄联系电话
  └── 备注
  ↓
点击"保存"
  ↓
系统验证表单（必填项检查）
  ├── 验证失败 → 提示错误信息
  └── 验证成功 → 生成项目编号（格式：PRJ20260127001）
                   ↓
               保存到数据库
                   ↓
               初始化流程状态为"新项目"
                   ↓
               记录流程日志
                   ↓
               跳转到项目详情页
```

**详细步骤**：
1. 用户在项目列表页点击"新建项目"按钮
2. 打开项目创建表单
3. 填写必填信息：
   - 项目类型：新建项目/改造项目/扩建项目
   - 城市：支持搜索和选择
   - 项目名称（站名）
   - 项目地址
   - 邮寄地址
   - 邮寄联系人
   - 邮寄联系电话
4. 填写可选信息：备注
5. 点击"保存"按钮
6. 系统前端验证表单：
   - 检查必填项是否填写
   - 验证电话号码格式
7. 后端再次验证
8. 生成项目编号：PRJ + 日期（YYYYMMDD）+ 三位序号
9. 插入projects表：
   - created_by = 当前用户ID
   - current_flow_step = new_project
10. 插入project_flow_logs表：
    - flow_step = new_project
    - status = completed
11. 记录operation_logs
12. 返回项目ID，跳转到项目详情页

### 流程3：创建冷库并添加设备

**参与角色**：系统管理员、企业管理员

```
进入项目详情页
  ↓
点击"新建冷库"
  ↓
填写冷库信息
  ├── 冷库名称
  ├── 冷库类型（下拉选择）
  ├── 面积
  ├── 高度
  └── 备注
  ↓
系统自动计算
  ├── 容积 = 面积 × 高度
  └── 制冷剂量 = 容积 × 系数
  ↓
保存冷库
  ↓
进入冷库详情页
  ↓
添加设备（冷风机、温控器、电表、机组、冷柜）
  ↓
选择设备类型
  ↓
填写设备信息
  ├── 品牌（下拉选择）
  ├── 型号（联动品牌）
  ├── 数量
  ├── 出厂编号
  └── 备注
  ↓
系统自动生成设备编号（AC-001-001）
  ↓
保存设备
  ↓
重复添加其他设备
```

**详细步骤**：

#### 3.1 创建冷库
1. 用户在项目详情页点击"新建冷库"
2. 填写表单：
   - 冷库名称（必填）
   - 冷库类型（必填，下拉选择：低温库/冷藏库/中温库）
   - 面积（必填，数字，单位：平方米）
   - 高度（必填，数字，单位：米）
   - 备注（可选）
3. 选择冷库类型后，系统自动填充设计温度范围：
   - 低温库：-18℃ ~ -15℃
   - 冷藏库：1℃ ~ 4℃
   - 中温库：8℃ ~ 10℃
4. 用户输入面积和高度后，系统实时计算：
   - 容积 = 面积 × 高度
   - 制冷剂量 = 容积 × 系数（低温0.5，冷藏0.3，中温0.2）
5. 点击"保存"
6. 系统验证并插入cold_rooms表
7. 更新项目流程为"设备登记"（如果还是"新项目"）
8. 跳转到冷库详情页

#### 3.2 添加设备
1. 在冷库详情页，选择要添加的设备类型（冷风机/温控器/电表/机组/冷柜）
2. 点击"添加设备"按钮
3. 填写设备表单：
   - 品牌（必填，下拉选择，来自标准设备库）
   - 型号（必填，下拉选择，根据品牌联动）
   - 数量（必填，默认1）
   - 出厂编号（可选，用户手动填写）
   - 备注（可选）
4. 选择型号后，系统自动填充：
   - 通讯端口类型（如RS485）
   - 通讯协议（如Modbus RTU）
5. 系统自动生成设备编号：
   - 规则：设备类型缩写-冷库ID-三位序号
   - 例如：AC-001-001（第一个冷库的第一个冷风机）
6. 点击"保存"
7. 插入devices表
8. 返回冷库详情页，显示设备列表
9. 用户可以继续添加其他设备，或点击"复制"功能快速添加同类设备

### 流程4：建立设备关系

**参与角色**：系统管理员、企业管理员

```
进入冷库详情页
  ↓
点击"设备关系配置"
  ↓
选择关系类型
  ├── 冷风机-机组关系
  ├── 冷风机-温控器关系
  ├── 电表-设备关系
  ├── 冷柜-温控器关系
  └── 冷柜-机组关系
  ↓
选择源设备和目标设备
  ↓
保存关系
  ↓
系统生成关系记录
  ↓
在关系表中显示
```

**详细步骤**：
1. 用户在冷库详情页点击"设备关系配置"标签
2. 点击"添加关系"按钮
3. 选择关系类型（下拉选择）：
   - 冷风机 → 机组
   - 冷风机 → 温控器
   - 电表 → 设备
   - 冷柜 → 温控器
   - 冷柜 → 机组
4. 选择源设备（下拉选择，只显示符合类型的设备）
5. 选择目标设备（下拉选择，只显示符合类型的设备）
6. 填写关系描述（可选）
7. 点击"保存"
8. 系统验证：
   - 检查设备是否存在
   - 检查关系是否已存在（避免重复）
   - 检查约束（如一个冷风机只能对应一个机组）
9. 插入device_relationships表
10. 返回关系列表，显示新增关系
11. 用户可以查看关系表格或关系图（简单表格展示）

### 流程5：网关配置与端口分配

**参与角色**：系统管理员、企业管理员

```
进入项目详情页
  ↓
点击"网关管理"
  ↓
点击"添加网关"
  ↓
填写网关信息
  ├── 品牌
  ├── 型号（自动显示端口数）
  ├── 网关编号
  └── 备注
  ↓
保存网关
  ↓
系统自动生成端口列表（根据端口数）
  ↓
配置每个端口
  ├── 选择绑定设备（可通讯的设备）
  ├── 分配485地址（自动或手动）
  └── 保存端口配置
  ↓
更新项目流程为"网关及485通讯分配"
  ↓
导出网关配置清单
```

**详细步骤**：

#### 5.1 添加网关
1. 用户在项目详情页点击"网关管理"标签
2. 点击"添加网关"按钮
3. 填写表单：
   - 品牌（必填，下拉选择）
   - 型号（必填，下拉选择，联动品牌）
   - 网关编号（必填，用户输入）
   - 备注（可选）
4. 选择型号后，系统自动显示端口数量
5. 点击"保存"
6. 系统插入gateways表
7. 根据端口数量，自动插入gateway_ports表：
   - 端口号：1, 2, 3, ..., port_count
   - 状态：unassigned
8. 跳转到网关配置页面

#### 5.2 配置端口
1. 显示端口列表（Port 1, Port 2, ...）
2. 对于每个端口，用户可以：
   - 选择绑定设备（下拉选择，只显示有通讯功能的设备）
   - 分配485地址：
     - 自动分配：系统从1开始递增
     - 手动输入：用户输入，系统校验唯一性
3. 点击"保存"
4. 系统更新gateway_ports表：
   - device_id = 选择的设备ID
   - rs485_address = 分配的地址
   - status = assigned
5. 同时更新devices表的rs485_address和gateway_port_id字段
6. 更新项目流程为"网关及485通讯分配"
7. 用户可以导出网关配置清单（Excel或PDF）

### 流程6：项目流程推进

**参与角色**：系统管理员、企业管理员

```
进入项目详情页
  ↓
点击"流程管理"
  ↓
查看当前流程状态
  ↓
点击"推进到下一步"
  ↓
系统更新流程状态
  ├── 标记当前节点为完成
  ├── 更新项目当前流程
  └── 记录流程日志
  ↓
显示流程进度条
```

**流程节点顺序**：
1. 新项目
2. 设备登记
3. 网关及485通讯分配
4. 网关登记
5. 网关邮寄
6. 通讯验证
7. 平台上线
8. 功能确认及完成

**详细步骤**：
1. 用户在项目详情页点击"流程管理"标签
2. 显示流程进度条，标识当前节点
3. 用户点击"推进到下一步"按钮
4. 系统弹出确认对话框："确认推进到【下一节点名称】？"
5. 用户确认
6. 系统执行：
   - 插入project_flow_logs：当前节点状态为completed
   - 更新projects表的current_flow_step为下一节点
   - 插入project_flow_logs：下一节点状态为started
   - 记录operation_logs
7. 刷新页面，显示新的流程状态
8. 如果到达"功能确认及完成"，项目标记为已完成

## 三、异常流程处理

### 3.1 登录失败锁定
- **场景**：用户连续5次输入错误密码
- **处理**：锁定账号30分钟，提示"账号已锁定，请30分钟后再试"
- **解锁**：30分钟后自动解锁，或系统管理员手动解锁

### 3.2 删除项目确认
- **场景**：用户点击删除项目
- **处理**：弹出二次确认对话框："确认删除项目【项目名称】？此操作将删除所有相关数据且不可恢复。"
- **结果**：用户确认后执行级联删除

### 3.3 设备关系冲突
- **场景**：用户尝试为一个冷风机分配第二个机组
- **处理**：提示"该冷风机已连接到机组【机组编号】，一个冷风机只能连接一个机组"
- **建议**：先删除现有关系，再建立新关系

### 3.4 485地址冲突
- **场景**：用户手动输入的485地址已被使用
- **处理**：提示"该485地址已被设备【设备编号】使用，请更换地址"
- **建议**：使用自动分配功能

### 3.5 数据权限拒绝
- **场景**：企业管理员尝试访问其他企业的项目
- **处理**：返回403错误，提示"无权访问该资源"

## 四、业务规则总结

### 4.1 数据创建规则
- 项目编号：PRJ + YYYYMMDD + 三位序号（自动生成）
- 设备编号：设备类型缩写-冷库ID-三位序号（自动生成）
- 容积计算：面积 × 高度
- 制冷剂计算：容积 × 系数（低温0.5，冷藏0.3，中温0.2）

### 4.2 关系约束规则
- 一个冷风机只能连接一个机组
- 一个冷风机可以对应多个温控器
- 一个电表可以监控多个设备
- 一个冷柜可以对应一个温控器和一个机组

### 4.3 流程推进规则
- 流程按顺序推进，不可跳跃
- 每次推进记录操作人和时间
- 流程可查询历史记录

### 4.4 权限控制规则
- 系统管理员：无限制
- 企业管理员：仅操作created_by = 自己的项目
