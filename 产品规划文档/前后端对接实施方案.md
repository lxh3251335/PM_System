# 前后端对接实施方案

## 📋 目标
将现有的前端演示系统（基于LocalStorage）改造为真实可用的系统（调用后端API），支持边测试边优化。

## 🎯 实施策略

### **渐进式迁移**
不一次性改造所有页面，而是按业务流程逐步迁移：
1. 先迁移核心流程（项目创建 → 设备登记）
2. 每迁移一个模块立即测试
3. 发现问题立即优化
4. 逐步完善所有功能

### **双模式支持**
初期支持两种模式切换：
- **演示模式**：使用LocalStorage（无需后端）
- **生产模式**：调用真实API（需要后端）

通过配置文件快速切换，方便开发调试。

## 🔧 技术实施方案

### **第一步：环境准备**

#### 1.1 安装MySQL数据库
```bash
# Windows用户：下载安装MySQL 8.0+
# 创建数据库
CREATE DATABASE pm_system DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 创建用户
CREATE USER 'pm_user'@'localhost' IDENTIFIED BY 'pm_password123';
GRANT ALL PRIVILEGES ON pm_system.* TO 'pm_user'@'localhost';
FLUSH PRIVILEGES;
```

#### 1.2 配置后端环境
```bash
cd D:\projects\PM_System\backend

# 安装Python依赖
pip install -r requirements.txt

# 配置环境变量（复制.env.example为.env）
copy env.example .env

# 编辑.env，修改数据库连接信息
```

#### 1.3 初始化数据库
```bash
# 创建表结构并初始化数据
python init_data.py
```

#### 1.4 启动后端服务
```bash
# Windows
start.bat

# 或直接运行
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

#### 1.5 验证后端运行
浏览器打开：http://localhost:8000/docs
应该能看到Swagger API文档。

### **第二步：创建前端API工具层**

创建统一的API客户端，所有页面通过它调用后端：

**文件结构：**
```
demo/
├── assets/
│   └── js/
│       ├── api.js          # API客户端（新建）
│       ├── config.js       # 配置文件（新建）
│       └── app.js          # 现有文件
```

**api.js 功能：**
- 统一的HTTP请求封装
- 错误处理
- Loading状态管理
- Token管理（为后续认证做准备）

---

### **第三步：页面迁移顺序**

#### **阶段一：基础模块（P0）**
1. ✅ **项目列表页** (project-list.html)
   - 调用 `GET /api/projects` 获取项目列表
   - 支持筛选、搜索
   - 删除项目调用 `DELETE /api/projects/{id}`

2. ✅ **项目创建页** (project-create.html)
   - 调用 `POST /api/projects` 创建项目
   - 调用 `POST /api/projects/{id}/copy` 复制项目

3. ✅ **冷库创建页** (cold-room-create.html)
   - 调用 `POST /api/projects/{id}/cold-rooms` 创建冷库
   - 支持批量创建

#### **阶段二：设备管理（P0）**
4. ✅ **设备登记页** (device-register.html)
   - 调用 `GET /api/equipment/brands` 获取品牌
   - 调用 `GET /api/equipment/models` 获取型号
   - 调用 `POST /api/devices` 创建设备
   - 调用 `POST /api/devices/{id}/copy` 复制设备

5. ✅ **设备关系页** (device-relations.html)
   - 调用 `GET /api/devices` 获取设备列表
   - 调用 `POST /api/devices/relations` 创建关系
   - 调用 `GET /api/devices/relations` 查询关系

#### **阶段三：网关通讯（P1）**
6. ✅ **网关配置页** (gateway-setup.html)
   - 调用 `POST /api/gateways` 创建网关
   - 自动获取网关型号参数

7. ✅ **通讯配置页** (communication-setup.html)
   - 调用 `POST /api/gateways/{id}/comm-config` 配置通讯
   - 485地址分配

#### **阶段四：查看与管理（P1）**
8. ✅ **项目详情页** (project-detail.html)
   - 调用 `GET /api/projects/{id}` 获取项目详情
   - 调用 `GET /api/projects/{id}/devices` 获取设备列表
   - 支持在线编辑

## 🔄 数据流设计

### **当前（演示模式）**
```
用户操作 → 前端页面 → LocalStorage → 页面更新
```

### **目标（生产模式）**
```
用户操作 → 前端页面 → API客户端 → 后端API → MySQL → 返回数据 → 页面更新
```

### **过渡方案**
```
用户操作 → 前端页面 → API客户端（检测模式）
                           ├─ 生产模式 → 后端API → MySQL
                           └─ 演示模式 → LocalStorage
```

## 🚀 部署方案

### **开发环境**
- 前端：本地文件系统（双击HTML打开）
- 后端：localhost:8000
- 数据库：本地MySQL

### **测试环境**
- 前端：Nginx静态服务器
- 后端：Uvicorn + Supervisor
- 数据库：测试MySQL服务器

### **生产环境**
- 前端：CDN + Nginx
- 后端：Gunicorn + Nginx反向代理
- 数据库：MySQL主从复制
