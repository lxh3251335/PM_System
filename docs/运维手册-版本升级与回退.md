## 一、本地开发完成后提交代码

### 1.1 日常修 Bug / 加功能后提交

在本地电脑 PowerShell 中执行：

```powershell
cd D:\projects\PM_System

# 第一步：查看改了哪些文件
git status

# 第二步：添加所有改动
git add .

# 第三步：提交（写清楚改了什么）
git commit -m "修复: 用户管理页面创建中文用户名报错"

# 第四步：推送到 GitHub
git push
```

### 1.2 提交信息规范（建议）

| 前缀 | 用途 | 示例 |
|------|------|------|
| 修复: | 修 Bug | `修复: 通讯设置删除设备后无法重新绑定` |
| 新增: | 新功能 | `新增: 网关库存批量导入功能` |
| 优化: | 改进现有功能 | `优化: 项目列表支持按登记时间排序` |
| 调整: | UI/文案调整 | `调整: 系统名称改为冷库项目信息管理系统` |

### 1.3 打版本标签（发布重要版本时）

```powershell
# 给当前代码打一个版本号标签
git tag -a v1.0.1 -m "修复用户管理和通讯设置相关bug"

# 推送标签到 GitHub
git push origin v1.0.1
```

打了标签后，即使后续代码继续变动，也可以随时回退到这个标签对应的版本。

## 二、服务器端升级软件

### 2.1 标准升级流程（3 分钟完成）

SSH 连接到服务器后执行：

```bash
# ====== 第一步：备份数据库（必做！）======
cp /opt/PM_System/backend/pm_system.db /opt/PM_System/backend/pm_system.db.bak_$(date +%Y%m%d_%H%M%S)

# ====== 第二步：拉取最新代码 ======
cd /opt/PM_System
git pull

# ====== 第三步：如果有新的 Python 依赖，更新依赖 ======
cd /opt/PM_System/backend
source venv/bin/activate
pip install -r requirements.txt

# ====== 第四步：重启后端服务 ======
systemctl restart pm-system

# ====== 第五步：验证服务正常 ======
systemctl status pm-system
curl http://localhost:8000/health
```

### 2.2 一键升级脚本（推荐）

在服务器上创建升级脚本，以后每次只需运行一个命令：

```bash
cat > /opt/PM_System/upgrade.sh << 'SCRIPT'
#!/bin/bash
echo "=============================="
echo " 冷库项目信息管理系统 - 升级"
echo "=============================="

# 备份数据库
BACKUP_FILE="/opt/PM_System/backend/pm_system.db.bak_$(date +%Y%m%d_%H%M%S)"
if [ -f /opt/PM_System/backend/pm_system.db ]; then
    cp /opt/PM_System/backend/pm_system.db "$BACKUP_FILE"
    echo "[1/5] 数据库已备份: $BACKUP_FILE"
else
    echo "[1/5] 未找到数据库文件，跳过备份"
fi

# 拉取最新代码
cd /opt/PM_System
echo "[2/5] 正在拉取最新代码..."
git pull
if [ $? -ne 0 ]; then
    echo "!! 代码拉取失败，请检查网络或 Git 配置"
    exit 1
fi

# 更新依赖
cd /opt/PM_System/backend
source venv/bin/activate
echo "[3/5] 正在更新依赖..."
pip install -r requirements.txt -q

# 重启服务
echo "[4/5] 正在重启服务..."
systemctl restart pm-system
sleep 2

# 检查状态
echo "[5/5] 检查服务状态..."
if systemctl is-active --quiet pm-system; then
    echo ""
    echo "=============================="
    echo " 升级成功! 服务运行正常"
    echo "=============================="
else
    echo ""
    echo "!! 服务启动失败，正在回滚..."
    # 如果新代码导致启动失败，可以手动回退（见第三节）
    systemctl status pm-system
    exit 1
fi
SCRIPT

chmod +x /opt/PM_System/upgrade.sh
```

以后每次升级只需：

```bash
bash /opt/PM_System/upgrade.sh
```

## 三、版本回退

### 3.1 回退到上一次提交

如果刚升级发现有问题，立即回退：

```bash
cd /opt/PM_System

# 回退到上一个版本（代码回退，数据库不受影响）
git checkout HEAD~1

# 重启服务
systemctl restart pm-system
```

### 3.2 回退到指定版本标签

```bash
cd /opt/PM_System

# 查看所有版本标签
git tag -l

# 回退到指定版本（如 v1.0.0）
git checkout v1.0.0

# 重启服务
systemctl restart pm-system
```

### 3.3 回退到指定的某次提交

```bash
cd /opt/PM_System

# 查看提交历史（找到想回退的版本）
git log --oneline -20

# 输出示例：
# a1b2c3d 修复: 用户管理页面创建中文用户名报错
# e4f5g6h 新增: 通讯拓扑图功能
# i7j8k9l 调整: 系统名称改为冷库项目信息管理系统
# ...

# 回退到指定提交（用前面的哈希值）
git checkout e4f5g6h

# 重启服务
systemctl restart pm-system
```

### 3.4 回退后想恢复到最新版本

```bash
cd /opt/PM_System

# 切回最新版本
git checkout main

# 重启服务
systemctl restart pm-system
```

### 3.5 数据库回退（极少需要）

如果升级后数据库结构有问题（极少发生），可以恢复备份：

```bash
# 查看有哪些备份
ls -la /opt/PM_System/backend/pm_system.db.bak_*

# 停止服务
systemctl stop pm-system

# 用备份替换当前数据库
cp /opt/PM_System/backend/pm_system.db.bak_20260224_153000 /opt/PM_System/backend/pm_system.db

# 重启服务
systemctl start pm-system
```

## 六、注意事项

1. **每次升级前必须备份数据库**（升级脚本已自动执行）
2. **数据库文件不在 Git 中**，`git pull` 不会覆盖数据
3. **前端是静态文件**，`git pull` 后立即生效，无需重启
4. **后端代码改动**需要 `systemctl restart pm-system` 才能生效
5. 如果服务器上本地修改过文件（如 config.js），`git pull` 可能冲突，用以下命令解决：
   ```bash
   git stash        # 暂存本地修改
   git pull          # 拉取最新代码
   git stash pop     # 恢复本地修改
   ```
