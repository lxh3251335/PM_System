# 数据库切换指南

## 📋 概述

本系统支持多种数据库，可根据不同阶段选择：
- **开发阶段**：SQLite（推荐）
- **测试阶段**：SQLite 或 MySQL
- **生产阶段**：MySQL 或 PostgreSQL

**切换非常简单，只需修改一行配置！**

---

## 🔄 切换方法

### 方法一：修改 .env 文件

编辑 `backend/.env` 文件：

```ini
# ============== SQLite配置 ==============
# 开发阶段推荐使用
DATABASE_URL=sqlite:///./pm_system.db

# ============== MySQL配置 ==============
# 生产阶段推荐使用
# DATABASE_URL=mysql+pymysql://pm_user:pm_password123@localhost:3306/pm_system

# ============== PostgreSQL配置 ==============
# 也可以使用PostgreSQL
# DATABASE_URL=postgresql://pm_user:pm_password123@localhost:5432/pm_system
```

**注意：** 取消注释你要使用的数据库，注释掉其他的。

### 方法二：使用环境变量（推荐用于部署）

**Windows PowerShell:**
```powershell
# 设置MySQL
$env:DATABASE_URL="mysql+pymysql://pm_user:pm_password123@localhost:3306/pm_system"

# 设置PostgreSQL
$env:DATABASE_URL="postgresql://pm_user:pm_password123@localhost:5432/pm_system"
```

**Linux/Mac Bash:**
```bash
# 设置MySQL
export DATABASE_URL="mysql+pymysql://pm_user:pm_password123@localhost:3306/pm_system"

# 设置PostgreSQL
export DATABASE_URL="postgresql://pm_user:pm_password123@localhost:5432/pm_system"
```

## 📊 数据库配置详解

### SQLite配置
```ini
# 语法：sqlite:///<数据库文件路径>
DATABASE_URL=sqlite:///./pm_system.db

# 绝对路径也可以
DATABASE_URL=sqlite:///D:/projects/PM_System/backend/pm_system.db

# 内存数据库（测试用，重启后数据消失）
DATABASE_URL=sqlite:///:memory:
```

**特点：**
- ✅ 无需安装任何软件
- ✅ 一个文件就是一个数据库
- ✅ 备份=复制文件
- ⚠️ 不适合高并发写入

### MySQL配置
```ini
# 语法：mysql+pymysql://<用户名>:<密码>@<主机>:<端口>/<数据库名>
DATABASE_URL=mysql+pymysql://pm_user:pm_password123@localhost:3306/pm_system

# 远程MySQL服务器
DATABASE_URL=mysql+pymysql://pm_user:pm_password123@192.168.1.100:3306/pm_system

# 使用域名
DATABASE_URL=mysql+pymysql://pm_user:pm_password123@db.example.com:3306/pm_system

# 添加字符集和其他参数
DATABASE_URL=mysql+pymysql://pm_user:pm_password123@localhost:3306/pm_system?charset=utf8mb4
```

**特点：**
- ✅ 高并发支持
- ✅ 企业级稳定性
- ✅ 丰富的管理工具
- ⚠️ 需要安装和配置

### PostgreSQL配置
```ini
# 语法：postgresql://<用户名>:<密码>@<主机>:<端口>/<数据库名>
DATABASE_URL=postgresql://pm_user:pm_password123@localhost:5432/pm_system

# 远程服务器
DATABASE_URL=postgresql://pm_user:pm_password123@192.168.1.100:5432/pm_system
```

**特点：**
- ✅ 功能最强大
- ✅ 严格的SQL标准
- ✅ 适合复杂查询
- ⚠️ 学习成本较高

## 🚀 切换步骤

### 从SQLite切换到MySQL

#### 1. 安装MySQL
参考：`backend/环境准备指南.md`

#### 2. 创建数据库和用户
```sql
CREATE DATABASE pm_system DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'pm_user'@'localhost' IDENTIFIED BY 'pm_password123';
GRANT ALL PRIVILEGES ON pm_system.* TO 'pm_user'@'localhost';
FLUSH PRIVILEGES;
```

#### 3. 修改配置
编辑 `backend/.env`:
```ini
# 注释掉SQLite
# DATABASE_URL=sqlite:///./pm_system.db

# 启用MySQL
DATABASE_URL=mysql+pymysql://pm_user:pm_password123@localhost:3306/pm_system
```

#### 4. 初始化MySQL数据库
```powershell
cd D:\projects\PM_System\backend
python init_data.py
```

#### 5. 重启后端服务
```powershell
# 停止当前服务（Ctrl+C）
# 重新启动
start.bat
```

#### 6. 验证
访问 http://localhost:8000/docs 确认服务正常。

### 从SQLite切换到PostgreSQL

步骤类似，只是：
1. 安装PostgreSQL
2. 创建数据库和用户
3. 修改配置为PostgreSQL连接字符串
4. 重新初始化和启动

## 💾 数据迁移

### 简单方法：重新初始化

**适用于：** 测试数据，可以丢弃

```powershell
# 1. 切换数据库配置
# 2. 重新初始化
python init_data.py
```

### 导出导入方法

**适用于：** 需要保留现有数据

#### SQLite → MySQL

```powershell
# 1. 导出SQLite数据为SQL文件
sqlite3 pm_system.db .dump > backup.sql

# 2. 编辑backup.sql，调整MySQL兼容性（如果需要）

# 3. 导入到MySQL
mysql -u pm_user -p pm_system < backup.sql
```

#### 使用Python脚本迁移（推荐）

创建 `migrate_data.py`:
```python
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker
from app.models import *

# 源数据库（SQLite）
source_engine = create_engine('sqlite:///./pm_system.db')
SourceSession = sessionmaker(bind=source_engine)
source_db = SourceSession()

# 目标数据库（MySQL）
target_engine = create_engine('mysql+pymysql://pm_user:pm_password123@localhost:3306/pm_system')
TargetSession = sessionmaker(bind=target_engine)
target_db = TargetSession()

# 迁移数据
try:
    # 迁移用户
    users = source_db.query(User).all()
    for user in users:
        target_db.merge(user)
    
    # 迁移项目
    projects = source_db.query(Project).all()
    for project in projects:
        target_db.merge(project)
    
    # ... 迁移其他表 ...
    
    target_db.commit()
    print("数据迁移完成！")
except Exception as e:
    target_db.rollback()
    print(f"迁移失败：{e}")
finally:
    source_db.close()
    target_db.close()
```

运行迁移：
```powershell
python migrate_data.py
```

## 🎯 选择建议

### 开发阶段
```
推荐：SQLite ⭐⭐⭐⭐⭐
理由：
- 零配置，立即开始
- 快速迭代
- 简单备份（复制文件）
- 足够测试所有功能
```

### 测试阶段
```
推荐：SQLite 或 MySQL
理由：
- 功能测试：SQLite足够
- 性能测试：使用MySQL
- 压力测试：使用MySQL
```

### 生产阶段
```
推荐：MySQL 或 PostgreSQL ⭐⭐⭐⭐⭐
理由：
- 高并发
- 企业级稳定性
- 专业运维工具
- 数据安全保障
```

### 特殊场景

**边缘设备/嵌入式：**
- 推荐：SQLite
- 理由：轻量级，单文件

**数据分析/BI：**
- 推荐：PostgreSQL
- 理由：强大的查询能力

**高并发Web应用：**
- 推荐：MySQL
- 理由：成熟的生态系统

**微服务架构：**
- 推荐：PostgreSQL 或 MySQL
- 理由：支持分布式事务

## 🔐 安全建议

### SQLite
- ✅ 文件权限控制
- ✅ 定期备份文件
- ⚠️ 不要直接暴露在网络

### MySQL/PostgreSQL
- ✅ 使用强密码
- ✅ 不要使用root用户
- ✅ 配置防火墙
- ✅ 启用SSL连接
- ✅ 定期备份
- ✅ 监控异常访问

## 📝 最佳实践

### 1. 开发流程
```
本地开发（SQLite）
    ↓
提交代码
    ↓
CI/CD测试（SQLite或MySQL）
    ↓
预发布环境（MySQL）
    ↓
生产环境（MySQL/PostgreSQL）
```

### 2. 配置管理
```python
# 在config.py中
import os

DATABASE_URL = os.getenv(
    "DATABASE_URL",
    "sqlite:///./pm_system.db"  # 默认SQLite
)
```

### 3. 连接池配置
```python
# SQLite
engine = create_engine(
    DATABASE_URL,
    connect_args={"check_same_thread": False}
)

# MySQL/PostgreSQL
engine = create_engine(
    DATABASE_URL,
    pool_size=20,
    max_overflow=40,
    pool_pre_ping=True
)
```
